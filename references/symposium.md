# 会饮与流觞实现规范

## 核心流程

角色始终并行出稿（半对话式），各自的回应作为中间产物保留为独立文件。在此基础上，AI将全部中间产物编织为一篇会饮记录——师傅们围坐讨论用户抛出的问题。

**关键规则：用户在会饮中是隐形的。** 师傅们围坐彼此讨论用户抛出的问题，用户是旁听者，不在场。角色提到用户时用第三人称（"这个年轻人""他/她"），不直接对着用户说话。中间产物（半对话）中角色的"你"，在会饮录中转化为"他/她"。

编织时AI需要：
- 将角色的直接对话（对着用户的"你"）转化为讨论（对着彼此的"他/她"）
- 找到角色之间观点的自然碰撞点
- 安排合理的对话顺序（谁先说、谁接话、谁反驳）
- 保持每个角色的语气和人格
- 标注关键时刻（被说服、反驳、沉默、转向）

## 流觞：高成本替代

流觞与会饮的区别在于出稿阶段——角色串行发言，前人的话后人听到再说，对话过程本身已是会饮，整理为会饮记录即可。

触发词："流觞"/"接力"

### 流觞实现

使用 TeamCreate 创建会饮团队，角色串行发言。

发言顺序安排原则：
- 与问题距离最近的角色先说（热身）
- 观点最具颠覆性的角色放中后段（高潮）
- 最具整合力或最意外的角色收尾

执行流程：
1. TeamCreate 创建会饮团队
2. 第一个角色发言（基于用户问题）
3. SendMessage 将前人发言传给下一个角色
4. 下一个角色基于用户问题 + 前人所有发言产出回应
5. 重复直到所有角色发言完毕
6. AI将全部发言整理为会饮记录
7. 关闭team

流觞中角色可以：
- 接着前人的话头展开
- 反驳前人的观点
- 被前人的话触动，修正自己原本想说的
- 沉默（"对此我没有要说的"本身也是姿态）

## 会饮记录格式

```markdown
# 会饮录：{主题}

> {场景设定：今天来了谁，因为什么聚在一起}

**{角色A}** [AI选]
{角色A的发言，保持角色语气}

**{角色B}** [自荐]
{角色B的回应，可以接话、反驳、展开}

**{角色C}** [随机]
{角色C的发言}

...

---

> {散场：未尽之言，开放结尾}

## 附录
- **共振**：{多角色收敛的方向}
- **分歧**：{真实分歧，不抹平}
```

开场设置场景，标注每个角色来路。对话正文保留各自语气。散场留白。附录简要提炼共振与分歧，方便回顾。
