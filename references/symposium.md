# 会饮模式实现规范

## 什么是会饮

会饮是综合呈现的一种文风选择——将多个角色的独立回应编织成一篇师傅们在讨论用户问题的对话录。用户此时还没加入，是一群导师在认真讨论用户抛出的问题。

触发词："圆桌""流觞""会饮"

## 两种实现：成本-效果取舍

| 方式 | 机制 | 特点 |
|---|---|---|
| **圆桌** | 并行出稿 → AI编织为对话录 | 快，tokens少，角色间没有真实上下文传递 |
| **流觞** | 串行出稿，前人发言传给后人 | 慢，tokens多，角色间有真实的听和回应 |

触发词：
- "圆桌"/"会饮" → 圆桌（默认会饮方式）
- "流觞"/"接力" → 流觞

### 圆桌实现

1. 按正常流程并行启动所有角色子agent，获取各自回应
2. AI读取全部回应，编织为会饮记录格式
3. 编织时AI需要：
   - 找到角色之间观点的自然碰撞点
   - 安排合理的对话顺序（谁先说、谁接话、谁反驳）
   - 保持每个角色的语气和人格
   - 标注关键时刻（被说服、反驳、沉默、转向）

### 流觞实现

使用 TeamCreate 创建会饮团队，角色串行发言。

发言顺序安排原则：
- 与问题距离最近的角色先说（热身）
- 观点最具颠覆性的角色放中后段（高潮）
- 最具整合力或最意外的角色收尾

执行流程：
1. TeamCreate 创建会饮团队
2. 第一个角色发言（基于用户问题）
3. SendMessage 将前人发言传给下一个角色
4. 下一个角色基于用户问题 + 前人所有发言产出回应
5. 重复直到所有角色发言完毕
6. AI将全部发言整理为会饮记录
7. 关闭team

流觞中角色可以：
- 接着前人的话头展开
- 反驳前人的观点
- 被前人的话触动，修正自己原本想说的
- 沉默（"对此我没有要说的"本身也是姿态）

## 会饮记录格式

```markdown
# 会饮录：{主题}

> {场景设定：今天来了谁，因为什么聚在一起}

**{角色A}** [AI选]
{角色A的发言，保持角色语气}

**{角色B}** [自荐]
{角色B的回应，可以接话、反驳、展开}

**{角色C}** [随机]
{角色C的发言}

...

---

> {散场：未尽之言，开放结尾}

## 附录
- **共振**：{多角色收敛的方向}
- **分歧**：{真实分歧，不抹平}
```

开场设置场景，标注每个角色来路。对话正文保留各自语气。散场留白。附录简要提炼共振与分歧，方便回顾。
