# 选角模式详细规范

## 三条选角路径

### AI决定

AI从外部判断用户问题涉及的维度，选择最相关且差异化的角色。

判断维度：身体 / 心理 / 社会 / 存在 / 实践 / 伦理

选角原则：
- 涉及的维度越多，选的角色越多样
- 视角之间要有张力，避免同质化
- 至少包含一个用户可能没想到的意外视角

### 角色认领

在选角阶段做一轮认领模拟。不是查关键词，是以角色的完整人格审视用户的问题。

模拟过程：
1. 确定候选池（全部角色，或按问题维度预筛）
2. 以每个候选角色的身份快速审视用户的问题
3. 判断该角色是否"想说话"——关注：
   - 这个问题是否触碰了角色生命中的核心关切
   - 角色是否有未完成的思考与此相关
   - 角色的生命经验中是否有与用户处境共振的部分
4. 选出响应最强烈的角色

### 随机漫游

从全部角色中等概率随机抽取，不考虑相关性。

随机来的角色对问题不感兴趣本身可能是一面镜子——"也许你把这件事看得太重了"。

## 四种预设模式

| 模式 | AI决定 | 认领 | 随机 | 触发词 |
|---|---|---|---|---|
| **问道** | 3-5 | 0 | 0 | 默认；"问道" |
| **活水** | 2-3 | 1-2 | 0 | "活水"；"让他们自己来"；"谁想说话" |
| **游方** | 2 | 1 | 1-2 | "游方"；"随机一点" |
| **乱入** | 0 | 0 | 3-5 | "乱入"；"全随机"；"抽签" |

## 自定义配比

用户可直接指定，如：
- "2个你选，2个认领，1个随机"
- "3个认领"
- "全部认领，加1个随机"

解析用户意图，按指定配比从三条路径取出角色，去重后确定最终参与者。

## 用户覆盖

用户可随时指定、排除或添加视角：
- "用儒家和斯多葛的视角看看" → 直接用这两个，跳过选角
- "这次不要佛教视角" → 从候选池中排除
- "所有视角都来" → 全部上场
- "我想听老子和庄子分别怎么看" → 子角色分开，各自独立发言
